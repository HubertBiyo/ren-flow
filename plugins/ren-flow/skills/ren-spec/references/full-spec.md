# ren-spec · 全程起草参考

> 何时读这份:判规模为「全程(full spec)」、起草前读。快路(fast spec)不用读 —— SKILL.md 里的判规模与退出条件已够;直接起草 4 节规格让用户一次确认即可。

## 起草模板

按下面结构起草,frontmatter `status: draft`。**整稿一次性给 review**,不分批 —— 分批用户看不出跨节不一致。

```markdown
---
doc_type: spec
slug: {英文连字符}
status: draft | approved | done | archived
summary: {一句话需求}
mode: fast | full
---

## 1. 要做什么 / 为什么
真问题一句话。用户目标。**明确不做什么**（具体到能被反向核对，不写「不过度设计」这种空话）。

## 2. 方案设计
- **数据 / 类型变化**：值对象 / 实体 / 接口 / 类型,用「现状 → 变化」两段式,「现状」指向代码位置。
- **主流程**：关键编排 / 控制流。涉及分支或多步时配一张流程图建 mental model。
- **放在哪儿**：这次的东西归属哪个模块?现有模块扩展 / 抽公共层 / 新建模块 —— 写明理由,别默认往最顺手的文件塞。
- **流程级约束**：错误语义、幂等、并发 / 顺序、可观测点(按需写)。

## 2.5 结构健康度（强制小节，结论必须显式写出）
动笔实现前评估三件事，**哪怕结论是「不动」也要写出来** —— 不强制写，AI 默认就是不思考往最顺手的胖文件里塞代码：
- **要改的文件**：即将改动的文件是否已偏胖 / 职责混杂？
- **目标目录**：新文件要落进的目录是否已摊平拥挤？
- **可卸载性**：这个功能要拆掉的话得拔哪些地方？答不出 = 边界没想清，要重新想。

结论三选一：① 现状健康，不动（写明）；② 先做「只搬不改行为」的微重构（拆文件 / 移定义，编译器全程绿灯）——列为推进步骤第 1 步且独立验证；③ 结构问题大到超出「只搬不改」——提示后续走 `ren-refactor`，不阻塞本次。

## 2.6 接口契约（仅当本次新增 / 改对外接口时写）
本次涉及对外 HTTP 接口、且该域维护 `arch/openapi/{slug}.openapi.json` 时,在方案里列出**接口清单**:每个接口的 method + path + 关键请求 / 响应字段 + 鉴权 header。定稿后这份清单是 build 阶段 seed / 更新 openapi 契约文件的依据(供 Apifox 导入联调）。纯内部逻辑、不外联调的改动跳过本节。详见 `ren-arch` 的「openapi/ 接口契约文件」。

## 2.7 上线影响面（仅当本次需要代码之外的上线动作时写）
本次若需发新服务 / 改配置中心配置 / 加调度任务 / 跑 DB 脚本,在方案里一句话标出**有哪些类动作**(不展开具体值)。可执行清单细化留给 build 阶段产出 `{slug}-release.md`。纯代码、无上线副作用的改动跳过本节。

## 3. 验收契约
每条写成「输入 / 触发 → 期望可观察结果」,覆盖正常 + 边界 + 错误。含「明确不做」的反向核对项。不写测试代码 / framework。

## 4. 推进步骤
2-5 个步骤。每步满足:① 粒度够小 —— 能独立完成、独立验证,不是「实现整个功能」这种大块;② 有明确的「做完的标志」(能跑的测试 / 能观察的结果);③ 有依赖的步骤排前面。不下沉到 file:line —— 那是 ren-build 扫完现状才定。
```

## 起草纪律

- **别替用户做决定**：用户没说清的角落,写成「假设：……」让用户能精确反驳;给 2-3 个选项再讲倾向,不自己挑一个填上去。
- **目标写成可验证的**：不写「让它能跑」「体验顺畅」,改写成「输入 A 返回 B」。
- **术语先锁死**:新概念名动笔前 grep 代码 / 架构防冲突。
- **示例优先于定义**:接口行为先给「输入→输出」示例。
- **不写实现细节**:改动文件清单、函数级落点、测试代码 —— 留给 `ren-build` 扫完现状自决。

## 容易踩的坑

- 没读架构 / 没 grep 术语就动笔 —— 方案跟现有代码对不上
- 用散文描述接口,不给示例 —— 读者建不起模型
- 在规格里写测试代码 / 函数级落点 —— 归 ren-build 自决
- 只给半份文档先 review —— 用户看不出全局一致性
- 把大需求硬塞进一份 spec —— 该退回 `ren-roadmap` 先拆
- 小事也走全程 —— 30 分钟的事走快路或直接改
- 跳过第 2.5 节结构健康度 —— 反熵增的拦截点就在动笔前,跳过等于默认往胖文件塞
