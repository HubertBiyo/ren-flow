## 上下文压缩保留优先级（Compact Instructions）

> 对话变长触发自动压缩、或手动 `/compact` 时，按此保留信息。一丢就要返工的放最前。owner 按项目特点裁剪。

**必须完整保留（不得概括掉关键值）：**
- 当前作业坐标：正在改的目录 / 分支 / 部署目标
- 当前制品状态：spec / fix 的 slug、进度、落点（`.ren-flow/specs|fixes/...`）
- 已对齐的契约：接口字段名与类型、表结构、时间戳单位、集合 / 索引名等开工前确认过的决策
- 用户本会话给过的纠正、口径、"有意为之"的特例
- 未完成的验证：没跑通的 build / test 命令、失败报错原文、待修 bug 的现象与根因结论

**可以丢弃 / 压缩：**
- 读过且本次无改动的文件全文（留路径与结论）
- 已通过命令的完整输出（留"已通过"结论）
- 被否决方案的细节（留"为何否决"一句话）
